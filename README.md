# journal-portal-pc

这是一个用于归档和发布多个独立 HTML 页面的静态仓库。本仓库不针对任何特定域名或网站，仅作为页面资源的集中存放与版本管理使用。

## 项目简介

`journal-portal-pc` 旨在提供一个简洁、可维护的 HTML 页面集合。仓库中的每个页面均为独立文件，彼此之间无强耦合，适合用于展示、存档或分发各类静态内容。

## 目录结构

```
journal-portal-pc/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 公共资源文件（如 CSS、JS、图片等）
├── index.html      # 仓库入口页（可选）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件代表一个独立页面，文件名即页面标识。
- `assets/`：存放页面引用的公共资源，按需组织子目录。
- `index.html`：可选入口页，用于简单导航或说明。

## 页面归档说明

- 所有页面均为独立 HTML，可直接通过浏览器打开或部署至任意静态托管服务。
- 页面之间不共享状态或依赖，便于单独维护与更新。
- 归档内容不涉及具体域名、网站或商业推广，仅作为技术性页面集合。

## 维护说明

- 添加新页面：在 `pages/` 目录下新建 HTML 文件，并确保引用资源路径正确。
- 修改已有页面：直接编辑对应文件，保持页面结构与命名规范。
- 资源管理：公共资源建议放入 `assets/`，避免重复；页面私有资源可放在同目录下。
- 提交信息：请使用简洁清晰的 commit 描述，便于追溯变更。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/<your-username>/journal-portal-pc.git
   ```
2. 在 `pages/` 目录下查看或编辑页面。
3. 部署时，将整个仓库上传至静态服务器，或通过 GitHub Pages 等工具发布。
